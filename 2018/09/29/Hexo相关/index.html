发表于 2018-09-29

Hexo 简介

Hexo是一个快速、简介且高效的blog框架。主要使用Markdown（一种可以使用普通文本编辑器编写的标记语言，通过简单地标记，可以使普通文本具有一定的格式）解析文章

安装：

前提：电脑安装有Node.js 、Git（下载对应安装包安装即可）

使用nmp（JavaScript世界的包管理工具）安装:

$ npm install -g hexo-cli

Mac用户可能会遇到错误，一般来说是编译错误，在AppStore安装Xcode并安装其中的命令行工具（Command Line Tools）即可，或仅安装命令行工具也可以

配置：

需要在指定文件夹内建立所需文件（也就是说你想“安装”Hexo在哪里）

$ hexo init <folder>
$ cd <folder>
$ npm install

此文件夹内包含：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
│   └── _posts
└── themes

_config.yml 详见

网站的配置信息，可以在此配置大部分的参数。

package.json

Hexo应用程序的信息。

scaffolds

模版文件夹。当新建文章时，Hexo 会根据 scaffold 来建立文件。

themes

主题文件夹。Hexo 会根据主题来生成静态页面。

Hexo 常用命令

选项
$ hexo --safe 
//安全模式，不会载入插件和脚本。

$ hexo --debug
//调试模式，在终端显示调试信息并记录到 debug.log 中。

$ hexo --silent 
//简洁模式，隐藏终端信息。

$ hexo --config custom.yml
//自定义配置文件到路径，执行后不再使用 _config.yml 。

$ hexo --draft
//显示草稿，显示 source/_drafts 文件夹中的草稿文章。

$ hexo --cwd /path/to/cwd
//自定义当前工作目录的路径。
init
$ hexo init [folder]
//未指定 foledr 时，将在当前文件夹建立网站。
new
$ hexo new [layout] <title> 
//新建一篇文章。如果没有设置 layout 的话，默认使用 _cinfig.yml 中的 default_layout 参数代替。如果标题包含空格的话，须使用引号扩起来。
generate
$ hexo generate 
//生成静态文件  可简写为 
$ hexo -g

    选项 -d ， --deploy  文件生成后立即部署网站
         -w ， --watch   监视文件变动
publish
$ hexo publish [layout] <filename>
//发表草稿
server
$ hexo server
//启动服务器。默认情况下，访问网址为： http://localhost:4000/
    选项 -p ， --port 重设端口
         -s ， --static 只使用静态文件
         -l ， --log 启动日志记录，使用覆盖记录格式
deploy
$ hexo deploy 
//部署网站 可简写为
$ hexo d 
    参数 -g ， --generate 部署之前先预先生成静态文件
render
$ hexo render <file1>[file2] ...
//渲染文件
    参数 -o ， --output 设置输出路径
migrate
$ hexo migrate <type>
//从其它博客迁移内容
clean
$ hexo clean 
//清除缓存文件（db.json）和已生成的静态文件（public）。更换主题后不生效时，可能需要执行此命令。
list
$ hexo list <type>
//列出网站资料
version
$ hexo version 
//显示Hexo版本
